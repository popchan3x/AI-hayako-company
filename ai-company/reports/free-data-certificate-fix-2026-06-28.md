# 無料データ通信失敗の修正レポート（2026-06-28）

## 結論

Stooq と Yahoo の両方で出ていた `fetch failed` は、外部サイト停止ではなく、アプリ側のデータ取得処理が証明書確認で止まっていたことが原因でした。

対策として、通常の取得が証明書確認で失敗した場合だけ、Windows の信頼済み設定を使って再取得する処理を追加しました。

## 理由

- Node.js の取得処理では `UNABLE_TO_VERIFY_LEAF_SIGNATURE` が出ていました。
- 同じ URL は PowerShell の `Invoke-WebRequest` では HTTP 200 で取得できました。
- つまり、通信回線や Yahoo 側の全面停止ではなく、Node.js がこの環境の証明書を信頼できていない状態でした。

## 変更内容

- `src/dataProvider.js`
  - 通常取得に失敗した場合、証明書確認の失敗だけを判定する処理を追加しました。
  - 証明書確認の失敗だった場合だけ、Windows の信頼済み設定を使う再取得処理を追加しました。
  - 取得方法を警告文に残し、`node-fetch`、`windows-trust`、`test-fallback` のどれで取得したか分かるようにしました。
- `tests/analyzer.test.js`
  - 証明書確認で失敗した場合に、代替取得で Yahoo データを読めるテストを1件追加しました。

## 注意点

- Stooq が空のデータを返した場合は、推測せず Yahoo に切り替えます。
- 土日や休場日は、無料データ側の最新終値が止まることがあります。
- 無料データは遅延や欠損があるため、画面上では引き続き「検証用シグナル」として扱います。
- デモデータは本物の相場データではなく、画面や分析ロジックの動作確認用に作った仮の価格データです。実際の売買判断には使いません。
